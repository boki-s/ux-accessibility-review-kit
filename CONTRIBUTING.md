# Contributing

Thank you for helping improve the UX Accessibility Review Kit.

## Good checklist items

A strong review item should be:

- Specific enough to test on a real interface
- Written in plain language
- Useful for designers and developers
- Linked to accessibility, usability, clarity, or inclusive design

## How to suggest a change

1. Open an issue describing the problem or missing check.
2. If you can, propose the new checklist text.
3. Keep pull requests focused on one topic.

## Checklist format

Checklist items also live in `data/checklist.json` so they can be reused in other tools.

Each item should include:

- `id`: short stable identifier
- `category`: review category
- `title`: short label
- `description`: practical review guidance

## Tone

Use friendly, direct language. The goal is to help teams improve their work, not shame people for missing something.
