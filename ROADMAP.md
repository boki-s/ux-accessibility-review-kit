# Roadmap

This roadmap focuses on turning the kit from a simple checklist into a useful open-source review workflow for designers, developers, and small product teams.

## Near Term

- Add more form review checks for validation, helper text, and error recovery
- Add more keyboard and focus review checks
- Add more WCAG references where the mapping is direct and useful
- Improve mobile layout review guidance
- Add copy-to-clipboard support for Markdown reviews

## Next

- Add optional severity labels: low, medium, high
- Add example reviews for common UI patterns such as login forms, landing pages, dashboards, and contact forms
- Add a checklist coverage page showing which topics and WCAG criteria are represented
- Add saved project names for repeated reviews

## Later

- Create a Figma-friendly checklist version
- Add a browser extension prototype
- Add WCAG 2.2 quick reference pages
- Add community-submitted checklist packs for specific industries
- Publish a lightweight package or embeddable widget if there is community demand

## Maintainer Focus

The project should stay practical, readable, and beginner-friendly. New features should help people review real interfaces faster without turning the kit into a complex audit platform.

## Completed

- Add WCAG references to checklist items
- Add Serbian and English checklist variants
- Add review notes for each checklist item
- Add export to Markdown for completed reviews
- Add an example login form review
