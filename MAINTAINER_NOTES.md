# Maintainer Notes

UX Accessibility Review Kit should stay practical, lightweight, and easy to understand for people who are still learning accessibility.

## Content Principles

- Prefer plain-language guidance over specialist jargon
- Make each checklist item testable on a real interface
- Include WCAG references when they help, but do not make the UI feel like legal documentation
- Keep examples grounded in common product screens and portfolio work
- Support designers, developers, educators, and small teams equally

## Contribution Review

When reviewing a contribution, ask:

- Is the proposed check specific enough to act on?
- Would a junior designer or early-career developer understand it?
- Does it help catch a common issue before launch?
- Does it overlap with an existing item?
- Should it include a WCAG reference?

## Release Notes

Small documentation and checklist improvements can ship as patch releases. New workflow features, exports, translations, or checklist packs should ship as minor releases.

## Community Direction

The project is most useful when it becomes a shared review habit, not a complex audit platform. The maintainer focus is to keep the tool simple enough for quick reviews while making the underlying guidance more trustworthy over time.
